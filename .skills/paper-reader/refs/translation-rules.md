# 翻译规范细则

## 一、翻译范围

| 部分 | 是否翻译 | 说明 |
|---|---|---|
| Title | ✅ 翻译 | 中文标题 + 保留英文原标题 |
| Authors | ❌ 不译 | 保留原文 |
| Abstract | ✅ 翻译 | |
| 正文各 Section | ✅ 翻译 | 从 Introduction 到 Conclusion |
| 公式 | ❌ 不译 | LaTeX 原样保留 |
| 引用标记 | ❌ 不译 | [12], \cite{...} 等原样保留 |
| 图表标题 | ✅ 翻译 | Figure/Table caption 翻译 |
| 图表内容 | ❌ 不译 | 坐标轴标签、数字等保持原样 |
| 代码块 | ❌ 不译 | 保持原样 |
| References | ❌ 不译 | 标注"参考文献略" |
| 致谢 | ❌ 不译 | |

## 二、术语处理规则

### 首次出现

```
中文译法（English Term）
```

示例：
- 大语言模型（Large Language Model, LLM）
- 检索增强生成（Retrieval-Augmented Generation, RAG）

### 后续出现

直接使用中文译法，不再标注英文。

### 无公认译法

保留英文，括号加简短注释：
- Transformer（一种基于自注意力的神经网络架构）
- softmax（将向量归一化为概率分布的操作）

## 三、常见翻译陷阱

### 3.1 被动语态

英文论文大量使用被动语态，中文翻译应转为主动：

- ❌ 这个方法被提出了
- ✅ 本文提出了这个方法

### 3.2 长定语从句

英文长定语从句应拆分为短句：

- ❌ 我们提出了一种能够在大规模数据集上实现高效训练的同时保持模型性能的方法
- ✅ 我们提出了一种新方法。这种方法能在大规模数据集上高效训练，同时保持模型性能

### 3.3 代词指代

英文用 it/they/this 指代前文，中文翻译时应明确指代对象：

- ❌ 它可以处理多种任务
- ✅ 该模型可以处理多种任务

### 3.4 数学表达的文字描述

- "x is defined as y" → "x 定义为 y"
- "let f(x) denote..." → "令 f(x) 表示..."
- "we have the following equation" → "得到以下等式"

## 四、格式保留

### 4.1 列表

原文有编号列表，译文保持相同编号：
```
1. 第一项
2. 第二项
```

### 4.2 强调

原文 *斜体* 或 **加粗**，译文保持相同格式。

### 4.3 章节层级

保持原文的章节编号和层级：
```
## 3. 方法（Method）
### 3.1 模型架构（Model Architecture）
```
